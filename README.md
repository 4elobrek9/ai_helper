# AI Helper: Профессиональная Сводка и Руководство по Развертыванию

Данный документ представляет собой **профессиональную** сводку для проекта `ai_helper`. На основе предоставленного исходного кода (`main.py`, `ai_test.py`, `command.py`, `command_OCR.py`) и списка зависимостей (`requirements.txt`), мы предоставляем точное описание архитектуры, требований и процесса развертывания.

---

## 1. Обзор Проекта

`AI Helper` — это автономное приложение для голосового взаимодействия, предоставляющее функционал ИИ-помощника. Ключевая архитектурная особенность заключается в использовании **локально развернутых больших языковых моделей (LLM)** через платформу `Ollama`, что исключает зависимость от внешних облачных сервисов. Этот подход гарантирует **полную конфиденциальность данных** пользователя и обеспечивает **автономность работы**, что критически важно для чувствительных приложений или сред с ограниченным сетевым доступом.

### 1.1. Ключевые Архитектурные Решения и Преимущества

* **Локальная LLM Обработка:** Все вычисления, связанные с ИИ (включая генерацию ответов), выполняются непосредственно на рабочей станции пользователя. Это минимизирует задержки и устраняет риски, связанные с передачей данных через публичные сети.

* **Императив Конфиденциальности:** Поскольку данные не покидают периметр пользовательской системы, обеспечивается максимальный уровень конфиденциальности и соответствие локальным политикам безопасности.

* **Голосовое Управление и Активация:** Использование `pvporcupine` обеспечивает **обнаружение "горячего слова" ("wake word")** для активации ассистента, а `SpeechRecognition` отвечает за распознавание речи пользователя.

* **Гибкость Моделей:** Архитектура проекта, основанная на `Ollama`, позволяет легко конфигурировать и переключаться между различными моделями LLM. В текущей реализации используется **модель `llama3`** с программно заданным промптом, который имитирует "Saiga-подобную" персону (добрая, милая девушка с интересной жизнью).

* **Модульная Расширяемость:** Проект демонстрирует хорошую модульную структуру, выделяя отдельные компоненты для:

    * **Синтеза речи (`sintez.py`):** Преобразование текстовых ответов ассистента в голосовые.

    * **Взаимодействия с графическим интерфейсом (`command_OCR.py`):** Автоматизация действий мышью на основе распознавания текста на экране с использованием **OCR (Optical Character Recognition)**. Реализовано с использованием `easyocr` (для английского) и `pytesseract` (для русского), что обеспечивает надежное распознавание текста на скриншотах, дополненное нечетким сопоставлением (`fuzzywuzzy`) для повышения точности.

    * **Выполнения внешних команд (`command.py`):** Интеграция с внешними сервисами, в частности, для получения актуальных данных о погоде. Механизм включает автоматическое определение местоположения по IP и парсинг данных с `open-meteo.com` (с резервным вариантом через Яндекс.Погоду).

* **Аудио-визуальная Обратная Связь:** Используются звуковые сигналы (`pygame`) для обозначения различных состояний ассистента (активация, подтверждение, ошибка).

* **Тестовый Модуль (`ai_test.py`):** Предоставляет отдельный скрипт для тестирования и демонстрации взаимодействия с локальной нейросетью Ollama, а также потенциального использования резервных облачных LLM (через `g4f`, хотя в текущей версии не используется).

## 2. Предварительные Требования к Развертыванию

Для успешного развертывания и функционирования `AI Helper` необходимо обеспечить наличие следующих системных компонентов и программного обеспечения:

1.  **Ollama (Версия 0.1.20 или выше):** Фундаментальная платформа для управления и запуска локальных LLM.

2.  **Модель `llama3`:** Основная языковая модель, используемая проектом. Должна быть загружена через Ollama.

3.  **Python (Версия 3.8.x - 3.12.x):** Основной интерпретатор для выполнения логики приложения. Рекомендуется использовать поддерживаемые версии для совместимости с библиотеками.

4.  **pip (Актуальная версия):** Стандартный менеджер пакетов Python, необходимый для разрешения и установки проектных зависимостей.

5.  **Tesseract OCR (исполняемый файл):** **Критически важен** для работы модуля `command_OCR.py` (распознавание русского текста). Это отдельная программа, которую необходимо установить в вашей системе.

## 3. Процесс Установки и Настройки

Следующий пошаговый план описывает процедуру развертывания `AI Helper` на целевой системе.

### 3.1. Установка Ollama

Ollama поддерживает кроссплатформенное развертывание на macOS, Linux и Windows.

* **macOS:**
    Загрузите дистрибутив по ссылке: [ollama.com/download/mac](https://ollama.com/download/mac).
    Запустите `.dmg` инсталлятор и следуйте инструкциям мастера установки.

* **Linux:**
    Откройте терминал и выполните команду для автоматизированной установки:

    ```bash
    curl -fsSL [https://ollama.com/install.sh](https://ollama.com/install.sh) | sh
    ```

* **Windows:**
    Загрузите исполняемый файл инсталлятора с: [ollama.com/download/windows](https://ollama.com/download/windows).
    Запустите `.exe` файл и завершите установку, следуя указаниям.

**Верификация Установки Ollama:**
После завершения установки, подтвердите работоспособность Ollama, выполнив в терминале:

```bash
ollama --version
```

Ожидаемый вывод: номер версии Ollama, что подтверждает успешное развертывание службы.

### 3.2. Загрузка Модели LLM (`llama3`)

После успешной установки Ollama, необходимо загрузить целевую языковую модель `llama3`.

В терминале выполните команду загрузки:

```bash
ollama pull llama3
```

Время загрузки зависит от пропускной способности вашего интернет-соединения и размера модели. Для ознакомления с полным каталогом доступных моделей посетите [ollama.com/library](https://ollama.com/library).

### 3.3. Установка Tesseract OCR

Для корректной работы OCR-функционала с русским языком необходимо установить исполняемый файл Tesseract OCR.

* **Windows:**
    Загрузите инсталлятор с официальной страницы [Tesseract GitHub](https://tesseract-ocr.github.io/tessdoc/Downloads.html). Рекомендуется выбирать версию с поддержкой русского языка (`tessdata/rus.traineddata`).
    После установки **крайне важно** убедиться, что путь к исполняемому файлу `tesseract.exe` прописан в системной переменной PATH или что путь к нему жестко указан в коде `command_OCR.py` (как показано в примере `pytesseract.pytesseract.tesseract_cmd = r'C:\Program Files\Tesseract-OCR\tesseract.exe'`).

* **macOS:**
    Установите через Homebrew:

    ```bash
    brew install tesseract
    brew install tesseract-lang
    ```

* **Linux (Debian/Ubuntu):**

    ```bash
    sudo apt install tesseract-ocr
    sudo apt install tesseract-ocr-rus # для русского языка
    ```

### 3.4. Развертывание AI Helper (Python Проект)

`AI Helper` является стандартным проектом на Python.

1.  **Клонирование Исходного Кода:**
    Убедитесь, что `git` установлен. Затем, в выбранной рабочей директории, выполните клонирование репозитория (замените плейсхолдер на фактический URL, если он будет известен):

    ```bash
    git clone https://github.com/4elobrek9/ai_helper
    cd ai_helper
    ```

2.  **Инициализация Виртуального Окружения (Рекомендуемая Практика):**
    Для изоляции зависимостей проекта и предотвращения конфликтов с системными пакетами Python, создайте и активируйте виртуальное окружение:

    ```bash
    python -m venv venv
    ```

    * **Для Windows:**

        ```powershell
        .\venv\Scripts\activate
        ```

    * **Для macOS/Linux:**

        ```bash
        source venv/bin/activate
        ```

    *(Примечание: После активации виртуального окружения префикс `(venv)` должен появиться в вашей командной строке.)*

3.  **Установка Зависимостей Python:**
    Проект использует множество внешних библиотек. Установите все необходимые зависимости из файла `requirements.txt`:

    ```bash
    pip install -r requirements.txt
    ```

## 4. Эксплуатация Системы

После успешного выполнения всех шагов установки и настройки, `AI Helper` готов к запуску.

1.  **Подтверждение Работоспособности Ollama:** Убедитесь, что служба Ollama активна в фоновом режиме (она должна быть запущена автоматически после установки или может быть запущена вручную).

2.  **Запуск Приложения:**
    Перейдите в корневую директорию проекта `ai_helper` в активном виртуальном окружении и запустите основной исполняемый скрипт:

    ```bash
    python main.py
    ```

    После запуска в терминале будут отображаться логи работы ассистента.

### 4.1. Пример Взаимодействия

1.  **Ожидание активации:** Ассистент находится в режиме ожидания "горячего слова".

2.  **Активация:** Произнесите "Lumia" (или другое сконфигурированное "горячее слово", соответствующее `lumia.ppn`). Ассистент должен проиграть звуковой сигнал (`right.mp3`) и произнести "Слушаю вас".

3.  **Произнесение команды/вопроса:** После активации произнесите вашу команду (например, "какая погода?", "наведи на кнопку Сохранить" или "расскажи про историю машинного обучения").

4.  **Обработка:**

    * Если команда соответствует одной из предопределенных (`наведи на`, `нажми на`, `кликни на`, `клик по`, `погода`, `стоп`, `выход`), она будет выполнена. Команды "наведи на" и "нажми на" используют OCR для поиска текста на экране.

    * Если команда не распознана как предопределенная, ассистент отправит запрос к локальной модели `llama3` через Ollama, используя ваш вопрос как промпт.

    * После получения ответа, ассистент озвучит его.

```
[Пользователь]: Lumia
[AI Helper]: Слушаю вас
[Пользователь]: Нажми на кнопку "ОК"
[AI Helper]: (выполняет клик по тексту "ОК" на экране)
[Пользователь]: Расскажи мне кратко про историю машинного обучения.
[AI Helper (Saiga Persona)]: Секунду, формулирую мысль... Машинное обучение, область искусственного интеллекта, зародилось в середине 20 века...
```

## 5. Руководство по Разработке и Вкладу

Проект `AI Helper` приветствует вклад сообщества. Для участия в разработке следуйте стандартному рабочему процессу Git/GitHub:

1.  **Форкните** основной репозиторий проекта.

2.  **Создайте новую ветку** для ваших изменений, используя осмысленное имя (например, `feature/implement-ocr-drag`, `bugfix/fix-audio-timeout`).
    `git checkout -b feature/новая-функциональность`

3.  **Реализуйте** изменения и **зафиксируйте** их, используя атомарные и описательные сообщения коммитов, соответствующие конвенциям (например, Conventional Commits).
    `git commit -m "feat: Добавить функцию X для обработки Y"`

4.  **Отправьте** вашу ветку в ваш форк.
    `git push origin feature/новая-функциональность`

5.  **Создайте Pull Request (PR)** в основной репозиторий, подробно описывая внесенные изменения, их обоснование и результаты тестирования. Обеспечьте, что ваш код соответствует PEP 8 и прошёл все автоматизированные тесты, если таковые имеются.

### 5.1. Потенциальные Области для Улучшений

* **Конфигурация:** Внедрение файла конфигурации (например, YAML, JSON) для таких параметров, как Ollama URL, модель LLM, ключ доступа Porcupine, пути к аудиофайлам, путь к Tesseract OCR, а также параметры промптов для Ollama. Это повысит гибкость и упростит развертывание на разных машинах.

* **Обработка Ошибок:** Расширение механизмов обработки ошибок, особенно для сетевых запросов и взаимодействия с внешними API/модулями (например, более гранулярная обработка ошибок `requests` и `SpeechRecognition`).

* **Многопоточность:** Оптимизация управления потоками для обеспечения более отзывчивого и стабильного взаимодействия, возможно, с использованием пулов потоков или асинхронного программирования для I/O операций.

* **Динамическая Загрузка Команд:** Реализация механизма плагинов или динамической загрузки команд, позволяющего добавлять или изменять функционал без модификации основного кода `main.py`.

* **Уточнение Промптов LLM:** Текущий проект использует разные промпты для Ollama в `main.py` и `ai_test.py`. Рекомендуется унифицировать или обеспечить гибкую настройку промптов для различных сценариев использования.

* **Интеграция Резервного LLM:** Развитие логики `ask_ai` в `ai_test.py` для полноценного использования `g4f` как резервного варианта, если локальная Ollama недоступна или не отвечает.

## Сборка в обычный `.exe` (один файл) + иконка

1. Подготовьте иконку приложения:
   - для окна/трея: `app_icon.png` (или `icon.png`),
   - для exe (рекомендуется): `app_icon.ico`.
2. Запустите сборку:

```bash
python build_exe.py
```

3. Готовый файл будет в `dist/AI_Helper.exe`.

Что уже сделано в проекте:
- иконка окна задаётся через `page.window.icon`,
- иконка в системном трее задаётся через `pystray.Icon(...)`,
- если есть `app_icon.ico`, он автоматически используется как иконка EXE при сборке.


См. также пошаговую проверку в `TESTING.md`.
